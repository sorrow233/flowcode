# 核心 10 大优先功能 (Top 10 Priority Features)

基于 `FUTURE_BETA.md`，我为你精心挑选了 10 个最应该优先实现的功能。
**选择逻辑**: 优先考虑能带来"视觉冲击(Wow Factor)"、"核心实用性"以及"良好的用户体验"的功能。

## 🎨 视觉与交互体验 (Polished UX & Visuals)

### 1. 全局命令面板 (Cmd+K)
- **来源**: ⌨️ 方向七 #1
- **理由**: 这是现代开发工具的标配，能瞬间提升网站的"专业感"和"高级感"。用户可以通过键盘快速导航，极具极客范儿。
- **实现**: 集成 `cmdk` 库，索引所有路由和功能。

### 2. 页面过渡动画 (Page Transitions)
- **来源**: 🎨 方向二 #8
- **理由**: 消除页面跳转的生硬感，让整个应用感觉像是一个流动的整体，极大增强"日式美学"的沉浸感。
- **实现**: 使用 `framer-motion` 在路由切换时添加淡入淡出或滑块效果。

### 3. 自定义主题配色器 (Theming)
- **来源**: 🎨 方向二 #3
- **理由**: 允许用户切换主色调（如朱赤、藤紫），不仅符合"日式美学"的多样性，也是展示前端能力的绝佳方式。
- **实现**: CSS 变量 + localStorage 存储用户偏好。

### 4. 骨架屏加载状态 (Skeleton Loading)
- **来源**: 🎨 方向二 #4
- **理由**: 拒绝简陋的"Loading..."文字，用平滑闪烁的骨架屏代替，这是区分"业余作品"和"商业级产品"的重要细节。
- **实现**: 创建通用的 `<Skeleton />` 组件。

### 5. 吐司通知组件 (Toast Notifications)
- **来源**: 🎨 方向二 #6
- **理由**: 现在的 `alert()` 太原始了。我们需要优雅的、从角落滑入的通知来反馈操作结果（如"复制成功"）。
- **实现**: 使用 `sonner` 或自建 Context 管理通知队列。

## 🛠️ 核心实用功能 (Core Utility)

### 6. i18n 架构 (Internationalization)
- **来源**: 🌐 方向三 #1
- **理由**: 既然目标是 SEO 和多语言，这是必须尽早打下的地基。越晚做，重构成本越高。
- **实现**: 引入 `react-i18next`，建立语言文件结构。

### 7. 代码片段一键复制 (Copy Code)
- **来源**: 🎯 方向一 #3
- **理由**: 作为教程网站，用户最频繁的操作就是复制命令。没有这个按钮会非常令人生气。
- **实现**: 简单的 `navigator.clipboard` API 集成到 `CodeBlock` 组件。

### 8. 进度追踪系统 (Progress Tracking)
- **来源**: 🎯 方向一 #1
- **理由**: 给用户成就感，鼓励他们从头读到尾。看到进度条涨满是非常爽的心理反馈。
- **实现**: 利用 localStorage 记录已访问的页面/步骤。

### 9. 响应式侧边导航抽屉 (Mobile Drawer)
- **来源**: 🎨 方向二 #1
- **理由**: 确保移动端体验同样完美。目前的侧边栏在手机上可能体验不佳，需要一个流畅的抽屉式导航。
- **实现**: 移动端隐藏侧边栏，通过汉堡菜单唤出抽屉。

## 🧩 开发者工具 (Dev Tool)

### 10. Cloudflare API Token 验证器
- **来源**: 🔧 方向四 #1
- **理由**: 这是一个真正的"工具"功能，解决用户教程中"不知道 Token 对不对"的痛点，体现 FlowCode 不仅仅是文档，还是工具。
- **实现**: 前端直接 fetch 调用 CF 接口 `/verify` 端点。
